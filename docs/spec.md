Xの有力アカウントのトレードをミラトレするBOT
現時点のたたき台では、MT5 とつなぎ、指定した 8 銘柄の売値と買値を 0.5 秒ごとに表示する。
OpenClaw と連携し、通常は `app/main.py` の起動中に X の指定アカウントを軽く監視した結果を受け取れるようにする。
OpenClaw の前段で X の本文だけを軽く取得し、その本文一覧を OpenClaw に渡して解釈させるようにする。
本文取得は `https://syndication.twitter.com` の埋め込みタイムラインを使い、BrowserRelay や画像解析は常用しない。
通常運用では `app/main.py` の起動中に約 30 秒ごとに軽い監視を回し、新着投稿があったときだけ OpenClaw を呼ぶ。
起動直後の最初の 1 回は、直近投稿を既読扱いにして誤発注を避ける。
OpenClaw の判定は単なる強気弱気の分類ではなく、投稿者が実際に行っている売買をできるだけミラトレすることを基準にする。
単なる感想や事後コメントでは発火させず、実際の売買意図や継続保有が読める場合だけ受け取るようにする。
受け取りは `jsonl` の inbox で行い、mirrorbot 側では SQLite に保存する。
SQLite には少なくとも ユーザー、銘柄、方向、in日時、in価格、out日時、out価格 を持たせる。
mirrorbot 画面の監視ループ内で inbox を取り込み、自動売買が有効なら MT5 に新規発注する。
mirrorbot 画面下部には、最近読み込んだ投稿とその判定結果や見送り理由も表示する。
同じユーザーと同じ銘柄で反対方向のシグナルが来た場合は、既存建玉を先に決済してから新しい方向で入る。
自動売買の開始停止は画面上から切り替えられるようにする。
自動売買の数量は GUI から設定できるようにし、共通数量と銘柄ごとの数量上書きを持てるようにする。
各銘柄の数量が空欄なら共通数量を使い、共通数量も空欄ならその銘柄の最小数量を使う。
OpenClaw の各回の分析結果は Discord の `ミラトレbot` チャンネルにも送る。
各銘柄には、選んだ時間足で直近 30 本のチャートを表示する。
上段は USDJPYm、EURUSDm、JP225m、USOILm。
下段は XAUUSDm、XAGUSDm、BTCUSDm、ETHUSDm。
一般的な大きさのウインドウでも、上段4つ・下段4つで収まりやすい見た目にする。
各枠の上部では、銘柄名と同じ行に BID と ASK を表示する。
各枠の BID と ASK は銘柄名より目立つ見た目にする。
時間足は 1分、5分、15分、30分、1時間、4時間、日足 を全銘柄一括で切り替えられるようにする。
各チャートは、8 銘柄エリア内で個別に拡大と縮小ができるようにし、操作は小さなアイコンボタンで行う。
画面全体は暗い見た目で統一する。
保有中の建玉価格、逆指値、指値は、対応する銘柄のチャート上に重ねて表示する。
これらの線は目立ちすぎない点線にし、縦の表示範囲は通常の値動きを基準にする。
画面上部の余計な見出しは置かず、そのぶんをチャート表示に使う。
チャートは上下の余白を減らして見やすくする。
画面下部には、MT5 風のポジション一覧を 11 列で表示する。
画面下部には、残高、有効証拠金、必要証拠金、余剰証拠金、証拠金維持率も表示する。
